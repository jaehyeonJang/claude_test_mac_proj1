## Overview
개인 납세자가 자유 텍스트로 의뢰를 먼저 입력하면, AI가 법령 해석에 필요한 필드를 결정해 표시하고, 사용자가 해당 정보만 입력하여 절세 방안 보고서를 받는다.

## 시나리오

### 1. 의뢰 입력 후 AI가 관련 필드 결정
[상황] 사용자가 의뢰 입력란(1단계)이 표시된 화면에 있으면
[동작] 시스템이 의뢰 내용을 AI로 분석하여 필요한 필드만 표시된 2단계 화면으로 전환한다

성공 기준:
- [ ] 1단계 화면에 의뢰 입력란(textarea)과 "다음" 버튼이 표시된다
- [ ] 구체적인 의뢰 입력 후 "다음" 클릭 시 로딩 인디케이터가 표시되고 "다음" 버튼이 비활성화된다
- [ ] 로딩 완료 후 2단계로 전환된다
- [ ] 2단계 상단에 입력한 의뢰 내용이 읽기 전용으로 표시되며, 표시 텍스트가 1단계 입력 내용과 일치하고 편집할 수 없다
- [ ] 2단계에 AI가 결정한 관련 필드들과 "분석 요청" 버튼이 표시된다

### 2. 모호한 의뢰 입력 시 재입력 안내
[상황] 사용자가 의뢰 입력란(1단계)에 모호한 내용을 입력한 상태에서
[동작] 시스템이 더 구체적인 입력을 요청하는 안내 메시지를 표시한다

성공 기준:
- [ ] "절세 알려줘"처럼 상황 파악이 불가한 의뢰 후 "다음" 클릭 시 1단계가 유지된다
- [ ] "의뢰 내용이 모호합니다. 구체적인 상황을 입력해주세요" 문구가 정확히 표시된다
- [ ] 입력란이 편집 가능한 상태로 유지되고 기존 입력 내용이 그대로 남아 있다

### 3. 2단계 필드 입력 후 분석 요청 → 보고서 표시
[상황] 사용자가 AI가 결정한 필드가 표시된 2단계 화면에 있으면
[동작] 시스템이 의뢰 + 입력 필드를 합쳐 절세 방안 보고서를 생성한다

성공 기준:
- [ ] 필드 입력 후 "분석 요청" 클릭 시 로딩 인디케이터가 표시된다
- [ ] 분석 완료 후 보고서 영역에 절세 방안 내용이 표시된다
- [ ] 보고서에 [조문 보기] [해석 보기] 탭 버튼이 표시된다

### 4. 보고서 조문/해석 탭 전환
[상황] 보고서 화면에 [조문 보기] [해석 보기] 탭이 표시되고 기본 탭이 [해석 보기]인 상태
[동작] 시스템이 선택한 탭에 해당하는 내용을 표시한다

성공 기준:
- [ ] 기본 선택 탭은 [해석 보기]이며, LLM이 작성한 해석 텍스트가 표시된다
- [ ] [조문 보기] 탭 클릭 시 관련 법령 조문(조문 제목 + 원문)이 표시된다
- [ ] [해석 보기] 탭 클릭 시 다시 LLM 해석 내용으로 전환된다

### 5. 채팅으로 추가 질문 → 보고서 재생성
[상황] 보고서가 표시된 화면 하단에 채팅 입력란이 있는 상태
[동작] 시스템이 추가 질문을 반영하여 보고서 전체를 재생성한다

성공 기준:
- [ ] 채팅 입력란에 텍스트 입력 후 전송 시 로딩 인디케이터가 표시된다
- [ ] 재생성 완료 후 이전 보고서가 새 보고서로 교체된다
- [ ] 채팅 메시지 내역(질문 + "보고서가 업데이트되었습니다" 안내)이 채팅 영역에 표시된다

### 6. 히스토리 저장 및 재조회
[상황] 분석 요청이 완료되어 보고서가 표시된 상태
[동작] 시스템이 질의+결과 세트를 localStorage에 저장하고 히스토리 목록에 표시한다

성공 기준:
- [ ] 보고서 생성 직후 히스토리 목록에 해당 항목(요청 시각 + 의뢰 내용 앞 20자)이 추가된다
- [ ] 의뢰 내용이 20자를 초과하면 히스토리 레이블에 앞 20자만 표시된다
- [ ] 히스토리 항목 클릭 시 해당 시점의 입력값과 보고서가 복원된다
- [ ] 페이지 새로고침 후에도 히스토리 목록이 유지된다

### 7. 다크모드 토글
[상황] 화면 우상단에 다크모드 토글 버튼이 표시된 라이트모드 상태
[동작] 시스템이 토글 상태에 따라 전체 화면의 테마를 전환한다

성공 기준:
- [ ] 토글 클릭 시 배경이 어두운 색으로 전환되고 다시 클릭 시 밝은 색으로 복귀한다
- [ ] 다크모드 상태가 localStorage에 저장되어 새로고침 후에도 유지된다
- [ ] localStorage에 저장된 다크모드 값은 OS 설정보다 우선 적용된다

### 8. 분석 요청 중 로딩 인디케이터 표시
[상황] 사용자가 2단계에서 필드를 입력하고 "분석 요청" 버튼을 클릭한 직후
[동작] 시스템이 보고서 생성 전 단계별 로딩 인디케이터를 표시한다

성공 기준:
- [ ] "분석 요청" 클릭 후 보고서 표시 전까지 로딩 인디케이터가 화면에 표시된다
- [ ] 로딩 중 "분석 요청" 버튼이 비활성화된다

### 9. 채팅 전송 중 로딩 인디케이터 표시
[상황] 보고서 화면에서 채팅 입력란에 질문을 입력하고 전송한 직후
[동작] 시스템이 보고서 재생성 전 로딩 인디케이터를 표시한다

성공 기준:
- [ ] 채팅 전송 후 재생성 완료 전까지 로딩 인디케이터가 표시된다

### 10. 분석 완료 후 히스토리 목록 자동 추가
[상황] 보고서가 처음 생성된 직후
[동작] 시스템이 질의+결과 세트를 히스토리 목록에 즉시 추가한다

성공 기준:
- [ ] 보고서 생성 완료 직후 히스토리 목록에 요청 시각 + 의뢰 내용 앞부분 항목이 표시된다

### 11. 다크모드 초기값 OS 설정 연동
[상황] localStorage에 다크모드 설정이 없는 상태에서 페이지에 최초 진입하면
[동작] 시스템이 OS 다크모드 설정에 맞는 테마를 자동 적용한다

성공 기준:
- [ ] OS 다크모드 ON + localStorage 미설정 → 다크 테마로 표시된다
- [ ] OS 다크모드 OFF + localStorage 미설정 → 라이트 테마로 표시된다

## 범위

### 포함
- 자유 텍스트 의뢰 입력 → AI 필드 결정 → 분석 2단계 플로우
- AI 기반 동적 필드 결정 (Gemini 호출)
- 모호한 의뢰 안내 메시지 처리
- 국가법령정보시스템(law.go.kr) 연동
- 보고서 탭(조문 나열 / LLM 해석)
- 채팅 기반 추가 질문 → 보고서 전체 재생성
- 히스토리 localStorage 저장 및 재조회
- 다크모드 토글

### 제외
- 법인/사업자 대상 분석 — 개인에 집중하여 초기 범위 축소
- 세무 신고서 자동 작성 — 분석·추천만 제공
- 회원가입/로그인 — localStorage로 데이터 관리
- AI가 결정한 필드를 사용자가 추가/제거 — 이번 범위 외

## 전제 조건
- AI 분석용 LLM API(Gemini) 키 발급 완료
- 브라우저 localStorage 사용 가능 환경

## 미결정 사항
- AI 필드 결정 로딩 중 UX (spinner vs 단계 표시)
