## Overview
개인 납세자의 소득·자산 정보를 입력받아 대한민국 세법(국가법령정보시스템 API 기반)을 AI로 분석하고, 법령 조문과 LLM 해석을 함께 제공하는 절세 방안 보고서를 도출한다.

## 시나리오

### 1. 구조화된 폼으로 절세 분석 요청
[상황] 사용자가 7개 항목(소득 유형, 연간 소득, 부양가족 수, 주택 보유, 금융소득, 연금/퇴직소득, 기납부세액)이 표시된 입력 폼 화면에 있으면
[동작] 시스템이 폼 값을 바탕으로 절세 방안 보고서를 생성하여 표시한다

성공 기준:
- [ ] 7개 폼 필드(소득 유형 Select, 연간 소득 입력, 부양가족 수 입력, 주택 보유 Select, 금융소득 Select, 연금/퇴직소득 Select, 기납부세액 입력)가 화면에 표시된다
- [ ] "분석 요청" 버튼을 클릭하면 로딩 인디케이터가 표시된다
- [ ] 분석 완료 후 보고서 영역에 절세 방안 제목과 내용이 표시된다
- [ ] 보고서에 [조문 보기] [해석 보기] 탭 버튼이 표시된다

### 2. 자유 텍스트로 추가 의뢰 입력
[상황] 사용자가 폼을 채운 상태에서 자유 텍스트 입력란이 표시되어 있으면
[동작] 시스템이 폼 데이터와 자유 텍스트를 합쳐 분석 요청을 처리한다

성공 기준:
- [ ] 자유 텍스트 입력란(textarea)이 폼 하단에 표시된다
- [ ] 자유 텍스트 입력 후 "분석 요청" 클릭 시, 보고서에 해당 내용이 반영된 결과가 표시된다
- [ ] 자유 텍스트만 입력해도 분석 요청이 가능하다

### 3. 보고서 조문/해석 탭 전환
[상황] 보고서 화면에 [조문 보기] [해석 보기] 탭이 표시된 상태이면
[동작] 시스템이 선택한 탭에 해당하는 내용을 표시한다

성공 기준:
- [ ] 기본 선택 탭은 [해석 보기]이며, LLM이 작성한 해석 텍스트가 표시된다
- [ ] [조문 보기] 탭 클릭 시, 관련 법령 조문(조문 제목 + 원문)이 표시된다
- [ ] [해석 보기] 탭 클릭 시, 다시 LLM 해석 내용으로 전환된다

### 4. 채팅으로 추가 질문 → 보고서 재생성
[상황] 보고서가 표시된 화면 하단에 채팅 입력란이 있으면
[동작] 시스템이 추가 질문을 반영하여 보고서 전체를 재생성한다

성공 기준:
- [ ] 채팅 입력란에 텍스트 입력 후 전송 시, 로딩 인디케이터가 표시된다
- [ ] 재생성 완료 후 이전 보고서가 새 보고서로 교체된다
- [ ] 채팅 메시지 내역(질문 + "보고서가 업데이트되었습니다" 안내)이 채팅 영역에 표시된다

### 5. 히스토리 저장 및 재조회
[상황] 분석 요청이 완료되어 보고서가 표시된 상태이면
[동작] 시스템이 질의+결과 세트를 localStorage에 저장하고 히스토리 목록에 표시한다

성공 기준:
- [ ] 보고서 생성 직후 히스토리 목록에 해당 항목(요청 시각 + 소득 유형)이 추가된다
- [ ] 히스토리 항목 클릭 시, 해당 시점의 폼 입력값과 보고서가 복원된다
- [ ] 페이지 새로고침 후에도 히스토리 목록이 유지된다

### 6. 다크모드 토글
[상황] 화면 우상단에 다크모드 토글 버튼이 표시된 상태이면
[동작] 시스템이 토글 상태에 따라 전체 화면의 테마를 전환한다

성공 기준:
- [ ] 토글 클릭 시 배경이 어두운 색으로 전환되고, 다시 클릭 시 밝은 색으로 복귀한다
- [ ] 다크모드 상태가 localStorage에 저장되어 새로고침 후에도 유지된다

### 7. 분석 요청 중 로딩 인디케이터 표시
[상황] 사용자가 폼을 채우고 '분석 요청' 버튼을 클릭한 직후 화면에 있으면
[동작] 시스템이 보고서 생성 전 로딩 인디케이터를 표시한다

성공 기준:
- [ ] '분석 요청' 클릭 후 보고서 표시 전까지 로딩 인디케이터가 화면에 표시된다
- [ ] 로딩 중에는 '분석 요청' 버튼이 비활성화된다

### 8. 자유 텍스트만으로 분석 요청
[상황] 폼 필드가 기본값(미입력)인 상태에서 자유 텍스트 입력란이 표시되어 있으면
[동작] 시스템이 자유 텍스트만을 기반으로 분석 요청을 처리한다

성공 기준:
- [ ] "프리랜서 소득으로 절세 방법 알려줘" 자유 텍스트만 입력 후 분석 요청 시 보고서가 표시된다
- [ ] 폼 미입력 상태에서 자유 텍스트만 있어도 '분석 요청' 버튼이 활성화된다

### 9. 채팅 전송 중 로딩 인디케이터 표시
[상황] 보고서 화면에서 채팅 입력란에 질문을 입력하고 전송한 직후 화면에 있으면
[동작] 시스템이 보고서 재생성 전 로딩 인디케이터를 표시한다

성공 기준:
- [ ] 채팅 전송 후 재생성 완료 전까지 로딩 인디케이터가 표시된다

### 10. 분석 완료 후 히스토리 목록 자동 추가
[상황] 보고서가 처음 생성된 직후 화면에 있으면
[동작] 시스템이 질의+결과 세트를 히스토리 목록에 즉시 추가한다

성공 기준:
- [ ] 보고서 생성 완료 직후 히스토리 목록에 요청 시각 + 소득 유형 항목이 표시된다
- [ ] 소득 유형 "근로소득"으로 분석 시 히스토리에 "근로소득"이 표시된다

### 11. 다크모드 초기값 OS 설정 연동
[상황] localStorage에 다크모드 설정이 없는 상태에서 페이지에 최초 진입하면
[동작] 시스템이 OS 다크모드 설정에 맞는 테마를 자동 적용한다

성공 기준:
- [ ] OS 다크모드 ON + localStorage 미설정 → 다크 테마로 표시된다
- [ ] OS 다크모드 OFF + localStorage 미설정 → 라이트 테마로 표시된다

## 범위

### 포함
- 개인 납세자 대상 절세 방안 분석
- 구조화된 폼(7개 필드) + 자유 텍스트 입력
- 국가법령정보시스템 API(law.go.kr) 연동
- 보고서 탭(조문 나열 / LLM 해석)
- 채팅 기반 추가 질문 → 보고서 전체 재생성
- 히스토리 localStorage 저장 및 재조회
- 다크모드 토글

### 제외
- 법인/사업자 대상 분석 — 개인에 집중하여 초기 범위 축소
- 세무 신고서 자동 작성 — 분석·추천만 제공, 공식 신고 기능은 별도 프로젝트
- 회원가입/로그인 — 개인 기기 localStorage로 데이터 관리
- 세무사 연결/상담 예약 — 이번 범위 외

## 전제 조건
- 국가법령정보시스템 OpenAPI 키 발급 완료
- AI 분석용 LLM API(Claude/GPT 등) 키 발급 완료
- 브라우저 localStorage 사용 가능 환경 (비공개 모드 제외)

## 미결정 사항
- 히스토리 최대 저장 개수 (기본값 50개로 가정)
- 다크모드 초기값 (시스템 설정 따름으로 가정)
- 보고서 스트리밍 여부(생성 중 실시간 표시 vs 완료 후 일괄 표시)
